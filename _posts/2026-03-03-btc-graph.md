**btc-graph** is an **interactive Bitcoin learning graph** that I built as a **static site for GitHub Pages**.

I wanted a way to organize Bitcoin concepts as a dependency graph, so readers could move from fundamentals to more advanced topics without losing context.

Each concept is modeled as a node with:

- a stable identifier
- one specific topic
- suggested prerequisites
- 2 to 3 curated high-signal resources

I also added a **search-friendly library view**, which makes it easier to browse topics outside the graph itself.

Under the hood, I keep the source of truth in `content/nodes/*.json`, which gets validated and compiled into a static graph dataset for the frontend.
